Général

Glossaire :

Le glossaire permet de définir toutes les variables qui vont être utiliser lors de l'algorithme.

Syntaxe : 
        Glossaire :
Expression Régulière correspondante :

"^glossaire\\s?:?$"

Déclaration de variable :

Le glossaire permet de définir toutes les variables qui vont être utiliser lors de l'algorithme.

Syntaxe : 
    Type NomVariable Commentaire
Exemple : 
        Entier somme1 Cette variable permet de calculer la somme.
Expression Régulière correspondante :

"^(entier|r[ée]el|cha[îi]ne|caract[eè]re|bool[ée]en)\\s+([a-z]+\\w*)\\s+((?:\\w*\\s*)*)$"

Début de l'algorithme :

Ce mot clé permet de définir le début de l'algorithme.

Syntaxe : 
        Début :
Expression Régulière correspondante :

"^d[eé]but\\s?:?$"

Fin de l'algorithme :

Ce mot clé détermine la fin de l'algorithme.

Syntaxe : 
        Fin
Expression Régulière correspondante :

"^fin$"

Commentaire :

Ceci permet de placer des commentaires dans l'algorithme.

Syntaxe : 
        //
Exemple : 
        //Ceci est une ligne de commentaire
Expression Régulière correspondante :

"^//.*$"

Structures Conditionnelles

On entend par structure conditionnelle, la possibilité d'orienter un algorithme en fonction de choix particuliers. Les structures conditionnelles nous ouvrent de nouveaux horizons pour l'élaboration d'algorithmes. Cela représente une notion très importante qu'il faut pouvoir assimiler si l'on cherche à aller plus loin. Il en existe deux.

Si ... FinSi :

Elle permet d'exprimer une condition simple (binaire).

Syntaxe : 
        Si condition Alors
            Code_à_éxécuter_si_l'évaluation_de_la_condition_est_vrai
        FinSi
Exemple : 
        Si var_age < 18 Alors
            Afficher "Vous n'êtes pas encore majeur."
        FinSi
Expression Régulière correspondante :

"^si\\s(" + expression + ")\\salors$"
"^finsi$"

Voir aussi :
Si ... Sinon... FinSi

Si ... Sinon... FinSi :

Elle permet d'exprimer une condition simple (binaire).

Syntaxe : 
        Si condition Alors
            Code_à_éxécuter_si_l'évaluation_de_la_condition_est_vrai
        Sinon
            Code_à_éxécuter_si_l'évaluation_de_la_condition_est_fausse
        FinSi
Exemple : 
        Si var_age < 18 Alors
            Afficher "Vous n'êtes pas encore majeur."
        Sinon
            Afficher "Vous pouvez conduire... OU boire!"
        FinSi
Expression Régulière correspondante :

"^si\\s(" + expression + ")\\salors$"
"^sinon$"
"^finsi$"

Selon :

Cette forme conditionnelle permet de mettre en place des choix multiples.

Syntaxe : 
        Selon variable_evalué :
            cas(valeur_1):
                action_1
            cas(valeur_2):
                action_2
            ...
            cas(valeur_n):
                action_n
            défaut:
                action_si_aucune_évaluation_n'a_été_vrai
        FinSelon
Exemple : 
        Selon note :
            cas(note<10):
                Afficher "Recalé"
            cas(note<12):
                Afficher "Passable"
            cas(note<14):
                Afficher "Assez Bien"
            cas(note<16):
                Afficher "Bien"
            cas(note<18):
                Afficher "Très bien"
            cas(note<20):
                Afficher "Exelent"
            défaut:
                Afficher "Euh... plus de 20 ? Vous êtes un extra-terrestre?"
        FinSelon
Expression Régulière correspondante :

"^selon\\s?(" + expression + ")\\s?:?\\s?$"
"^cas\\s?(" + expression + ")\\s?:?\\s?$"
"^d[ée]faut\\s?:?\\s?$"
"^finselon\\s?$"

Structures Répétitives

Dans certains, on peut se retrouver à avoir besoin de faire plusieurs fois la même opération. Si elles ne sont pas nombreuses, on pourrait à la rigueur, dupliquer autant de fois que nécessaire le morceau de l'algorithme qui nous interresse. Cependant, on se rend compte que cette solution n'est pas idéale. L'algorithme peut prendre rapidement une taille assez importante et si le traitement dupliqué est complexe et qu'une correction doit être effectuée, il faut alors la reporter sur chaque copie (sans compter que dans certains cas, on ne connaît pas à l'avance le nombre d'itérations que l'on aura à faire). Tout cela amène la notion de structure répétitive (ou boucle). On en compte trois, mais comme pour les structures conditionnelles, on pourrait aussi bien en utiliser qu'une seule.

Tant Que :

Cette instruction évalue une condition et tant que celle si est vérifiée, les actions placées à l'interieur sont alors effectuée. Pour sortir de la boucle, il faut que la condition ne soit plus remplie.

Syntaxe : 
        TantQue (condition)
            action
        FinTantQue
Exemple : 
        tour = 5
        Afficher "Compte a rebours : "
        TantQue (tour > 0)
            Afficher tour
            tour = tour - 1
        FinTantQue
Expression Régulière correspondante :

"^tantque\\s+(" + expression + ")\\s?$"
"^fintantque$"

Répéter Jusqu'à :

Cette instruction répète une action jusqu'à ce que la condition soit remplie.

Syntaxe : 
        Répéter
            action
        Jusqu'à (condition)
Exemple : 
        Répéter
            Afficher "Saisir un nombre positif : "
            Saisir nombre
        Jusqu'à (nombre > 0)
Expression Régulière correspondante :

"^r[ée]p[ée]ter$"
"^jusqu[\\s']?[àa]\\s?(" + expression + ")\\s?$"

Pour :

Elle affecte une valeur de départ à une variable, exécute les actions contenues dans la boucle, et l'incrémente jusqu'à la valeur de fin. L'incrémentation est par défaut de 1, mais il est possible de la régler avec l'instruction "pas de".

Syntaxe : 
        Pour variable1 de valeur1 à valeur2 pas de valeur3 faire :
            action
        FinPour
Exemple : 
        Pour i de 1 à 10 pas de 1 faire :
            Afficher "Compteur"
            Afficher i
        FinPour
Expression Régulière correspondante :

"^pour\\s+(" + expression + ")\\s+de\\s+(" + expression + ")\\s+[àa]\\s+(" + expression + ")\\s+(\\[pas\\s+de\\s+(" + expression + ")\\]\\s+)?faire\\s?:?\\s?$"
"^finpour$"

Fonctions

Voici quelques fonctions d'entrée sortie.

Afficher :

Elle permet d'afficher une variable ou bien un texte dans la console d'exécution (Le texte doit etre entre guillemets).

Syntaxe : 
        Afficher texte_ou_variable_à_afficher
Exemple : 
        Afficher "Bonjour"
        Afficher variable_de_la_mort_qui_tue
Expression Régulière correspondante :

"^afficher\\s+(.*)\\s?$"

Saisir :

Elle permet de saisir une variable.

Syntaxe : 
        Saisir variable
Exemple : 
        Saisir variable_age_de_lutilisateur
Expression Régulière correspondante :

"^saisir\\s+(" + listePipeVariable() + ")\\s?$"

Imprimer :

Elle permet de d'imprimer.

Syntaxe : 
        Imprimer
Exemple : 
        Imprimer
Expression Régulière correspondante :

"^imprimer\\s+(" + listePipeVariable() + ")\\s?$"
